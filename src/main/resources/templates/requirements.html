    • Requerimientos Funcionales:

    Registro de Usuario:

    • Los usuarios deben poder crear cuentas proporcionando información básica como nombre de usuario, dirección de correo electrónico y contraseña.
    • Los usuarios deben recibir correos electrónicos de confirmación para activar sus cuentas.

    Autenticación de Usuario:

    Los usuarios deben poder iniciar sesión en sus cuentas utilizando su nombre de usuario y contraseña.

    Exploración de Listas de Reproducción:

    Los usuarios deben poder buscar y navegar por listas de reproducción filtradas por género musical.
    Deben poder ver detalles de las listas de reproducción, incluyendo canciones y votos.

    Votación en Listas de Reproducción:

    Los usuarios deben poder votar positiva o negativamente por las listas de reproducción.
    Solo se permite un voto por usuario por lista de reproducción.

    Búsqueda de Canciones:

    Los usuarios deben poder buscar canciones por nombre y ver en qué listas de reproducción están incluidas.

    Gestión de Géneros Musicales (Administrador):

    El administrador debe poder agregar, editar y eliminar géneros musicales.

    Gestión de Canciones (Administrador):

    El administrador debe poder agregar, editar y eliminar canciones, incluyendo detalles como título, artista y género.

    Requerimientos No Funcionales:

    Usabilidad:

    La interfaz de usuario debe ser intuitiva y fácil de usar para usuarios de diferentes niveles de habilidad.
    Los tiempos de carga de la página deben ser razonables para brindar una experiencia fluida.

    Seguridad:

    La información del usuario, como contraseñas y datos personales, debe almacenarse de manera segura utilizando técnicas de cifrado adecuadas.
    La autenticación y la autorización deben implementarse de manera segura para prevenir accesos no autorizados.

    Escalabilidad:

    La aplicación debe ser capaz de manejar un aumento en el número de usuarios y listas de reproducción sin degradar el rendimiento.

    Disponibilidad

    La página web debe estar disponible y accesible para los usuarios la mayor parte del tiempo, con un tiempo de inactividad planificado mínimo.

    Rendimiento:

    Las búsquedas y la carga de listas de reproducción deben realizarse de manera eficiente, incluso cuando hay una gran cantidad de datos.

    Compatibilidad:

    La página web debe ser compatible con varios navegadores web modernos y dispositivos, como computadoras de escritorio, tabletas y dispositivos móviles.

    Mantenibilidad:

    El código debe estar bien estructurado y documentado para facilitar futuras actualizaciones y mejoras.

    Cumplimiento Legal y Regulatorio:

    La página web debe cumplir con las leyes de protección de datos y privacidad relevantes, como el RGPD (si aplica).
